# 星盘工坊 — 产品级 UI 架构最终方案

> 状态：方案确认 | 2026-05-22
> 融合来源：UI 重新设计方案 + 星盘工坊 v2 产品级方案

## 一、产品定位

**星盘工坊**是面向小说、漫画、漫剧创作者的专业 IDE，核心价值：

1. **沉浸创作** — 用户在任何阶段都能专注故事，而非工具操作
2. **全流程覆盖** — 大纲→写作→分镜→漫画→漫剧，一体化创作管线
3. **AI 协作** — AI 是创作搭档，提供智能建议、续写、逻辑检测，以悬浮气泡形式呈现
4. **认知贴合** — 信息架构与创作者心智模型贴合（世界→剧情→输出）

### 用户心智模型

创作者思考方式：
- **世界 → 剧情 → 输出**：先建立世界观和角色，再展开剧情，最后产出漫画/漫剧
- 时常在阶段间跳跃（写作→大纲→漫画→写作）
- 需要即时参考（Wiki / 角色 / 地点 / 分镜）
- AI 不应该打断沉浸，而是辅助决策

## 二、信息架构（IA）

### 树状导航：世界 → 剧情 → 输出

```
📖 作品
├─ 世界观
│   ├─ 人物（角色卡片 + Wiki 条目）
│   ├─ 地点
│   ├─ 势力
│   └─ Wiki（知识库条目）
├─ 剧情
│   ├─ 大纲（树形结构）
│   ├─ 卷/章（层级章节）
│   └─ 分镜（章节→镜头拆解）
├─ 输出
│   ├─ 漫画（页面构建）
│   ├─ 漫剧（视频合成）
│   └─ 导出（EPUB/PDF/PNG/MP4）
```

### 数据模型对应

| 层级 | UI 组件 | 数据模型 |
|------|---------|---------|
| 作品 | Dashboard | Project / AstrolabeConfig |
| 世界观 | RightPanel（属性面板） | Character / Location / WikiEntry |
| 剧情 | OutlineEditor / ChapterEditor | OutlineNode / Chapter |
| 输出 | ComicPageBuilder / DramaPlayer | ComicPage / ComicPanel / SpeechBubble |

## 三、全局导航系统

### 三模式切换

顶部导航栏使用三个主模式，替代六个阶段标签：

```
星盘工坊 v2      创作 │ 视觉化 │ 演出
                 ●      ○       ○
```

| 模式 | 包含阶段 | 说明 |
|------|---------|------|
| **创作** | 大纲 + 写作 | 文字创作的核心流程 |
| **视觉化** | 分镜 + 漫画 | 从文字到画面的转换 |
| **演出** | 漫剧 | 有声漫剧的最终输出 |

### 各模式下的子阶段切换

切换模式后，主工作区内部有当前模式的子阶段标签：

```
创作模式：
  [大纲] [写作]
   ●      ○

视觉化模式：
  [分镜] [漫画]
   ●      ○

演出模式：
  [漫剧]
   ●
```

### 左侧面板

```
世界观 │ 剧情 │ 输出
  ●      ○     ○
```

左侧面板内容随选中的导航标签变化：
- **世界观**：人物、地点、势力、Wiki 树
- **剧情**：大纲、卷/章、分镜树
- **输出**：漫画页、漫剧序列树

### 右侧面板

上下文感知，始终显示当前选中内容的关联信息：
- 选中大纲节点 → 显示节点属性（标题、概要、目标字数）
- 选中章节 → 显示相关 Wiki 参考卡片 + 角色卡片
- 选中分镜镜头 → 显示镜头详情 + 生成预览
- 选中漫画页 → 显示页面属性

### 底部状态栏

窄条，显示：创作进度 / 当前章节·字数 / AI 状态

- 右侧 Wiki 参考卡片默认折叠，按需展开
- 工具栏在底部（减少对标题区的干扰）

│  镜头4   │  │                            │  │

### 2. 漫画 — 构建

```
┌─────────────────────────────────────────────┐
│  菜单栏                                      │
├────────────────────┬────────────────────────┤
│   页面画布（65%）    │    抽屉式工具（35%）      │
│                    │                        │
│  ┌─────┬─────┬──┐  │  [模板] [素材] [气泡]    │
│  │ 格1  │ 格2  │  │  ┌──────────────────┐  │
│  │     │     │  │  │ □ 2×2  □ 3×2      │  │
│  ├─────┴─────┤  │  │ □ 1大3小  □ 自由    │  │
│  │   格3     │  │  └──────────────────┘  │
│  │          │  │                        │
│  └──────────┘  │  ┌── 分镜素材 —───────┐  │
│                    │ │ [镜头1]✓ [镜头4]✓ │  │
│                    │ │ [镜头2]✓ [镜头5]  │  │
│                    │ │ [镜头3]  [镜头6]  │  │
│                    │ └─────────────────┘  │
│                    │                        │
│  [生成当前页]       │  ┌── 对话气泡 ────────┐  │
│                    │  │ □矩形 □椭圆 □云朵   │  │
│                    │  └─────────────────┘  │
├────────────────────┴────────────────────────┤
│  第3章 · 漫画 · 第2页/共5页  [编辑] [预览]      │
└─────────────────────────────────────────────┘
```

**交互流程：**

1. 从分镜素材库拖拽镜头到页面画布格子中
2. 右侧抽屉切换模板/素材/对话三个面板
3. 页面模板：2×2、3×2、1大3小、全页出血、自由布局
4. 填充后叠加对话气泡（矩形/椭圆/云朵/锯齿样式可选）
5. AI 渲染按钮：逐格生成或整页生成
6. 切换"预览模式"查看最终漫画页面效果

**数据模型：**

```typescript
interface ComicPage {
  id: string;
  chapterId: string;
  pageNumber: number;
  layout: '2x2' | '3x2' | '1big3small' | 'full' | 'custom';
  panels: ComicPanel[];
}

interface ComicPanel {
  id: string;
  shotId: string;           // 关联的分镜镜头
  imageUrl?: string;         // AI 生成的漫画图
  position: { row: number; col: number; rowSpan: number; colSpan: number };
  bubbles: SpeechBubble[];
}

interface SpeechBubble {
  id: string;
  text: string;
  style: 'rect' | 'ellipse' | 'cloud' | 'jagged' | 'none';
  speaker: string;
}
```

## 八、AI 协作系统

### 悬浮 AI 气泡

AI 不再以固定按钮呈现，而是跟随用户操作的浮动工具：

```
用户选中一段文字 →

        ┌──────────────────┐
        │ ✨ AI 助手       │
        │                  │
        │ [续写]           │
        │ [改文风]         │
        │ [增强情绪]       │
        │ [检查人设一致性]  │
        │ [引用 Wiki]      │
        └──────────────────┘
         ⋮
       光标位置
```

### 上下文感知

- 选中章节 / 镜头 → 右侧面板显示相关角色 / 地点 / Wiki
- 默认折叠，点击展开详情
- 不打断创作流

### AI 能力矩阵

| 阶段 | AI 功能 |
|------|--------|
| 大纲 | 生成大纲、结构优化、冲突检测 |
| 写作 | 续写、改文风、增强情绪、人设崩坏提醒、Wiki 上下文注入 |
| 分镜 | 章节拆解为分镜、镜头 prompt 优化 |
| 漫画 | 逐镜头生成、整页渲染、对话气泡排版 |
| 漫剧 | 语音合成、字幕对齐、背景音匹配 |

## 十一、与同人库的关系

同人库是独立于单个作品的存在（工作区级别），不归入"世界→剧情→输出"树。它在左侧面板作为一个顶层入口：

```
👤 同人库（独立入口，左侧面板顶部）
├─ 人物卡片
├─ 世界观卡片
├─ 物品卡片
└─ 势力卡片
```

引入流程：在创作模式中，选中角色节点 → 右侧面板显示"从同人库引入"按钮 → 弹出选择卡片 + AI 平行宇宙改编对话框。

## 十三、设计原则总结

1. **世界 → 剧情 → 输出**的信息架构
2. **创作 / 视觉化 / 演出**三模式全局导航
3. 每个阶段拥有专属布局，不为折中牺牲体验
4. AI 以悬浮气泡形式呈现，不打断创作流
5. 右侧面板上下文感知，按需显示
6. 铜金暖色 + 暗灰背景，去工程感
7. 16px 基础字号，适合长文阅读
