# AI 创作管线 — 设计规格

> 子系统 #4 | 状态：已确认 | 2026-05-21

## 定位

星盘工坊的核心价值载体。将 AI 深度嵌入从大纲到成品的**完整创作链路**，AI 作为"总导演"提出方案，人在每个节点拍板决策。串起所有子系统——Wiki 提供知识上下文，同人库提供人物素材，一致性引擎确保视觉统一，AI 集成层提供模型能力。

## 管线状态机

任何时候作者都可以回溯修改，下游自动感知上游变更：

```
  [大纲] ──确认──▶ [人物] ──确认──▶ [章节] ──确认──▶ [分镜]
    │                │                │                │
    └── 不满意 ──────┘                │                │
        回到大纲调整                   │                │
                      不满意 ─────────┘                │
                      回到人物调整                      │
                                        不满意 ────────┘
                                        回到分镜调整
```

## 阶段一：大纲生成

AI 角色：故事架构师。输入作品标题、类型、一句话梗概，输出结构化大纲树。

### 交互模式

```
作者输入：我想写一个三国背景的金融商战小说
         │
         ▼
AI 生成 3 个大方向提案：
  方向1：穿越者在三国建立现代金融体系
  方向2：诸葛亮是穿越的华尔街投行家
  方向3：三国货币战争——各国争夺铸币权
         │
         ▼ (作者选择方向2)
AI 展开大纲树：
  第一卷：隆中对 · 商业计划书
    ├─ 第1章：诸葛亮的穿越
    ├─ 第2章：第一桶金——贩卖新型农具
    ├─ 第3章：结交刘备——天使轮融资
    └─ …
  第二卷：赤壁之战 · 恶意收购
  第三卷：三分天下 · 市场格局
         │
         ▼ (作者调整：移动章节、合并卷、改写节点描述)
确认大纲 → 写入 outline/outline.json
```

### 数据模型

```typescript
interface Outline {
  id: string;
  title: string;
  premise: string;
  genre: string[];
  volumes: Volume[];
  createdAt: string;
  updatedAt: string;
  history: OutlineVersion[];
}

interface Volume {
  id: string;
  title: string;
  summary: string;
  order: number;
  chapters: OutlineChapter[];
}

interface OutlineChapter {
  id: string;
  title: string;
  summary: string;
  keyEvents: string[];
  charactersInvolved: string[];
  targetWordCount: number;
  order: number;
  status: 'planned' | 'writing' | 'done';
}
```

### AI 策略

| 操作 | 模型 |
|------|------|
| 生成大方向 | deepseek-reasoner |
| 展开大纲树 | deepseek-reasoner |
| 调整节点 | deepseek-chat（流式） |

## 阶段二：人物构建

AI 角色：选角导演。扫描大纲，生成角色档案，自动推断关系网。

### 交互模式

```
大纲确认后 → AI 扫描大纲，提取所有被提及的角色
         │
         ▼
AI 生成角色建议清单：
  ┌─────────────────────────────────────────┐
  │ 主要角色                                 │
  │ ├─ 诸葛亮 — 穿越者，前华尔街分析师       │
  │ ├─ 刘备 — 创业者，寻找投资               │
  │ └─ 曹操 — 竞争对手，恶意收购专家          │
  │                                         │
  │ 次要角色                                 │
  │ ├─ 关羽 — 运营总监                       │
  │ ├─ 张飞 — 市场推广（暴力营销）            │
  │ └─ 孙权 — 家族企业继承人                  │
  │                                         │
  │       [全部展开]  [逐个确认]              │
  └─────────────────────────────────────────┘
         │
         ▼ (作者逐个审核角色)
对于每个角色：
  ├─ 确认 AI 生成的描述，或手动修改
  ├─ 可选：从同人库引入卡片（平行宇宙改编）
  ├─ 触发角色设定图生成（子系统#5）
  └─ 确认 → 写入 characters/char-xxx.json
```

### AI 自动推断关系网

AI 分析大纲中人名共现，建议角色关系，作者审核确认后写入各角色的 relations 字段。

## 阶段三：章节撰写

AI 角色：主笔作家。Wiki 上下文全面启用，流式生成章节内容。

### 交互模式

```
作者选择一个大纲节点 → 点击 "AI 撰写"
         │
         ▼
AI 准备上下文（自动）：
  ├─ 大纲：当前章节在故事中的位置、本章概要
  ├─ 角色：本章出场角色的 Wiki 条目
  ├─ 地点：当前场景的 Wiki 描述
  ├─ 前情：上一章摘要（或前 3 章摘要）
  ├─ 规则：可能触发的世界观规则
  └─ 风格：作者之前确认的写作风格偏好
         │
         ▼
AI 流式生成章节 → 实时逐字展示在编辑器中
         │
         ▼
作者阅读、修改、润色 → 确认
         │
         ▼
触发 Wiki 自动提取（子系统#2）
  └─ AI 扫描新章节 → 建议新条目 → 作者审核 → 更新 Wiki
```

### 上下文窗口管理（长篇关键）

采用分层摘要策略应对长篇小说上下文窗口限制：

```
System Prompt 内容预算 (~4000 tokens)：
  1. 当前章节大纲 (10%)
  2. 关键角色信息 (30%)
  3. 当前场景地点 (10%)
  4. 前情摘要 (25%)
  5. 相关规则提醒 (10%)
  6. 写作风格指令 (15%)

长篇记忆管理：
  卷级摘要 (200字) ← 每写完一卷，AI 自动生成
  章级摘要 (100字) ← 每写完一章，AI 自动生成
  创作时注入：
    ├─ 当前卷摘要
    ├─ 前 3 章摘要（详细信息）
    └─ 更早章节 → 仅卷级摘要
```

## 阶段四：分镜拆解

AI 角色：分镜师/导演。将已完成章节拆解为分镜脚本。

### 数据模型

```typescript
interface Storyboard {
  id: string;
  chapterId: string;
  shots: Shot[];
  createdAt: string;
  updatedAt: string;
}

interface Shot {
  id: string;
  order: number;
  scene: string;
  framing: 'extreme-long' | 'long' | 'medium' | 'close-up' | 'extreme-close-up';
  angle: 'eye-level' | 'high-angle' | 'low-angle' | 'bird-eye' | 'dutch';
  characters: ShotCharacter[];
  dialogue: ShotDialogue[];
  props: string[];
  mood: string;
  notes: string;
}

interface ShotCharacter {
  characterId: string;
  pose: string;
  expression: string;
  designVersion: number;
}

interface ShotDialogue {
  speakerId: string;
  text: string;
  type: 'speech' | 'thought' | 'narration';
}
```

### 交互模式

```
作者选择一章 → 点击 "分镜拆解"
         │
         ▼
AI 分析章节内容 → 拆解为分镜列表
         │
         ▼
分镜列表（内嵌在编辑器中）：
  ├─ 每个镜头：景别、角度、场景、角色动作、对话
  ├─ 每个角色镜头关联设定图引用
  └─ 作者审核：调整顺序、拆分/合并镜头、修改对话、补充道具描述
         │
         ▼
确认分镜脚本
```

## 阶段五：漫画生成

AI 角色：漫画渲染引擎。以分镜脚本 + 角色设定图为输入，生成漫画画面。

### 交互模式

```
作者确认分镜脚本 → 点击 "生成漫画"
         │
         ▼
系统预处理（自动）：
  对每个镜头：
    1. 加载出场角色的设定图（指定表情/姿态变体）
    2. 组装 prompt：
       "漫画风格，
        场景：{{shot.scene}}
        构图：{{shot.framing}} {{shot.angle}}
        角色：{{character.name}}（表情：{{expression}}，姿态：{{pose}}）
        氛围：{{shot.mood}}
        画风：日式漫画"
    3. 以角色设定图为 reference_image
    4. 调用火山引擎生成
         │
         ▼
进度面板：
┌──────────────────────────────────────┐
│ 镜头 1/42  ✅ 完成                    │
│ 镜头 2/42  ⏳ 生成中…                 │
│ 镜头 3/42  ⏳ 排队中…                 │
│                                      │
│ 预计剩余时间：约 8 分钟                │
│          [暂停]  [取消]               │
└──────────────────────────────────────┘
         │
         ▼ (全部生成完成)
作者逐镜头审阅：
  ├─ ✅ 满意 → 保留
  ├─ 🔄 不满意 → 重新生成（可修改 prompt）
  └─ 📝 手动补充 → 外部工具修改后替换
         │
         ▼
确认整话 → 导出（PNG 序列 / PDF）
```

### 批量生成优化

并行 3-4 个请求流水线处理。同一角色在连续镜头中复用设定图。

## 阶段六：漫剧生成

AI 角色：动画导演。将漫画静态画面转化为有声漫剧视频。

**漫剧 = 漫画画面 + 时间轴 + AI 配音 + 简单动效 + 字幕**

### 生成流水线

```
1. 时间轴编排
   ├─ AI 根据对话长度自动计算每个镜头停留时间
   ├─ 转场类型（切/淡入淡出/滑动）AI 建议
   └─ 作者可在时间轴上拖拽调整

2. 配音合成
   ├─ 每个角色绑定 TTS 声线（火山引擎）
   ├─ 声线示例：
   │   诸葛亮 → "沉稳中年男声"
   │   刘备 → "温和中年男声"
   │   张飞 → "粗犷男声"
   │   旁白 → "磁性男中音"
   └─ 支持声线试听、调整语速/音调

3. 动效叠加
   ├─ 对话镜头：轻微呼吸式缩放（Ken Burns）
   ├─ 动作镜头：方向性平移
   ├─ 转场：交叉淡入淡出
   └─ 重点画面：静态定格

4. 音效与背景音乐
   ├─ 场景自动匹配环境音
   └─ AI 根据场景 mood 推荐 BGM

5. 字幕渲染
   ├─ 从分镜对话自动生成字幕
   └─ 时间轴自动对轨
```

### 导出格式

```
├─ MP4 (H.264, 1080p) — 标准视频
├─ GIF — 短视频片段
└─ MP3 — 纯音频版本（有声书模式）
```

## 管线全局能力

### 版本快照

每个阶段确认后自动保存快照，作者可随时回溯。Wiki 不回溯（Wiki 是知识积累，不应丢失）。

### 上下文传递链

```
大纲 ──生成──▶ 角色 ──提供──▶ 章节 ──依赖──▶ Wiki ──注入──▶ 下一章
                                    │
                                    └──▶ 分镜 ──驱动──▶ 漫画 ──驱动──▶ 漫剧
                                               │              │
                                               └── 设定图 ────┘
                                               (一致性引擎)
```
