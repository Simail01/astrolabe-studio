# AI Wiki 功能 — 设计规格

> 状态：已确认 | 2026-05-22

## 定位

为星盘工坊 Wiki 知识库注入 AI 能力，让 AI 作为编辑助理：每写完一章后自动扫描并建议新设定条目；随时补充已有条目的新属性；全卷完成后检查角色/道具/时间线的一致性矛盾；自动发现条目间的关系网。**核心原则：AI 提案，人拍板。** 没有一条数据在未经作者确认的情况下写入 Wiki。

---

## P0：AI 提取新条目

### 触发

章节编辑器工具栏增加"Wiki 提取"按钮，写完一章后点击。

### 流程

```
作者点击 "Wiki 提取"
      │
      ▼
组装上下文 → DeepSeek：
  1. 本章完整内容
  2. 现有 Wiki 索引（条目 ID + 标题 + 类型，避免重复提取）
  3. 提取指令（Prompt 模板）
      │
      ▼
AI 返回 JSON 建议列表
      │
      ▼
右侧 Wiki 面板显示 "建议队列"
      │
      ▼
作者逐条：确认 / 编辑后确认 / 拒绝
      │
      ▼
确认条目 → 写入 wiki/<type>/<id>.json
```

### 建议队列 UI

```
🆕 发现 5 条新设定

✓ [角色] 诸葛亮    置信度 0.95
  依据: "羽扇纶巾，身长八尺"
  [确认] [编辑] [拒绝]

✓ [地点] 卧龙岗    置信度 0.9
  ...

    [全部确认] [全部拒绝]
```

### 提取 Prompt 模板

```
你是一位小说编辑助理。请分析以下章节内容，提取新出现的设定要素。

## 已有 Wiki 条目（请勿重复提取）
{{existingEntries}}

## 本章内容
{{chapterContent}}

## 请提取
1. 新角色：姓名、外貌、性格、能力、身份
2. 新地点：名称、描述、所属势力
3. 新物品：名称、类型、能力、来源
4. 新事件：名称、参与者、地点、影响
5. 新规则：世界观设定补充

## 输出 JSON 格式
[{
  "type": "person|location|item|event|rule",
  "title": "条目名称",
  "summary": "一句话摘要",
  "content": "详细描述",
  "attributes": {"key": "value"},
  "confidence": 0.0-1.0,
  "evidence": "原文中支持此条目的句子"
}]

## 规则
- confidence < 0.5 的不提取
- 已有条目不要重复提取
- 每个条目必须有 evidence
```

### 状态

| 字段 | 说明 |
|------|------|
| confirmedByUser: false | AI 建议，待确认 |
| confidence: AI 返回的值 | 作者确认后变为 1 |
| sourceChapters: [本章ID] | 信息来源 |

## P1：AI 补充已有条目

### 触发

Wiki 条目详情下方"AI 补充"按钮。

### 流程

```
作者选中 "诸葛亮" → "AI 补充"
      │
      ▼
AI 搜索全文章节中所有提到 "诸葛亮" 及其别名的段落
      │
      ▼
对比现有属性 → 发现新增/变化
      │
      ▼
生成补充建议列表（标注来源章节）
      │
      ▼
作者逐条：追加 / 覆盖 / 忽略
```

### 补充建议 UI

```
📝 诸葛亮 — 发现 3 条补充

[+外貌] 新增: "面容清瘦，鬓角微霜"
  来源: 第15章第3段
  [追加] [覆盖] [忽略]

[+性格] 新增: "偶尔急躁，尤其在战事吃紧时"
  来源: 第22章第7段
  [追加] [忽略]

[🆕武器] 袖中短剑
  来源: 第18章第12段
  [添加为新属性] [忽略]
```

### 约束

- AI 不删除已有内容，只建议新增
- 每条建议标注原文出处（章节号）
- 同名不同人问题：AI 标记低置信度，提示作者判断

## P2：AI 一致性检查

### 触发

工具栏"一致性检查"按钮，支持选择范围（当前卷 / 全文）。

### 流程

```
选择范围 → AI 逐章扫描
      │
      ▼
对比 Wiki 记录的属性与各章节描述
      │
      ▼
发现矛盾 → 按严重程度分级
      │
      ▼
生成一致性报告 → 作者逐条处理
```

### 分级标准

| 等级 | 条件 | 示例 |
|------|------|------|
| 🔴 严重 | 生死/存在性矛盾 | 关羽第3章已死亡，第80章又出场 |
| 🟡 警告 | 属性前后不一致 | 第5章"黑发"，第150章"白发苍苍"，仅隔3年 |
| 💭 提示 | 时间线/位置可疑 | 第20章在成都，第21章在荆州，时间连续但位置不可能 |

### 一致性报告 UI

```
🔍 一致性检查报告（全文）

🔴 关羽 — 生死矛盾
   第3章确认死亡 → 第80章再次出场
   建议：修改第80章出场角色
   [标记已修复] [标记有意为之] [忽略]

🟡 诸葛亮 — 外貌矛盾
   第5章:"黑发" → 第150章:"白发苍苍"
   时间跨度仅3年
   建议：统一外貌描述或添加"早衰"原因
   [标记已修复] [标记有意为之] [忽略]

💭 诸葛亮 — 时间线可疑
   第20章(成都) → 第21章(荆州)
   建议：插入过渡章节或调整时间线
   [标记已修复] [标记有意为之] [忽略]
```

### 处理动作

| 动作 | 效果 |
|------|------|
| 标记已修复 | 消除报告项，作者自行修改对应章节 |
| 标记有意为之 | 在 Wiki 中补充解释原因（如"关羽是假死"） |
| 忽略 | 暂不处理，保留在报告中 |

## P3：智能关系发现

### 触发

Wiki 面板顶部"AI 分析关系"按钮。

### 流程

```
AI 扫描全文章节
      │
      ▼
分析条目间共现 + 互动模式：
  - 共同出场频率 → 关联强度
  - 对话称呼 → 关系类型（主公/丞相 → 上下级）
  - 行为互动 → 敌对/盟友
  - 叙述文本中的明确关系
      │
      ▼
按置信度分类呈现 → 作者确认
```

### 关系类型

| 类型 | 说明 |
|------|------|
| 师徒 | 教授/学习关系 |
| 结义 | 结拜兄弟 |
| 上下级 | 主从/命令关系 |
| 敌对 | 对抗关系 |
| 血缘 | 亲属关系 |
| 盟友 | 合作关系 |
| 恋人 | 爱情关系 |
| 其他 | 用户自定义 |

### 关系建议 UI

```
🔗 发现 12 条潜在关系

强关联 (≥0.8)
  诸葛亮 ← 师徒 → 姜维    0.92
  证据: 第95章 "亮以维为传人"
  关羽   ← 结义 → 张飞    0.88

中等关联 (0.5 ≤ c < 0.8)
  诸葛亮 ← 敌对 → 司马懿  0.72
  证据: 多次对阵，互称"老对手"

    [批量确认(强关联)] [逐条处理]
```

### 约束

- 所有关系必须人工确认
- 每条建议标注证据（原文出处）
- 关系双向同步：添加 A→B，B 侧自动出现反向

## 技术实现要点

### 公共模式

所有四个功能共享相同的架构：

```
React 组件 → bridge → IPC → Main 进程 AIService
                                    │
                              DeepSeek API
                                    │
                              返回结果 → 解析 → 展示确认 UI
```

### Prompt 管理

四个功能的 Prompt 模板放在 `packages/ai/src/prompts/wiki/` 下：

```
wiki/
├── extract.txt      # P0 提取新条目
├── enrich.txt       # P1 补充已有条目
├── consistency.txt  # P2 一致性检查
└── relations.txt    # P3 智能关系发现
```

### Wiki 条目确认状态

新增 `WikiSuggestion` 类型用于 AI 建议队列：

```typescript
interface WikiSuggestion {
  id: string;
  type: WikiEntryType;
  title: string;
  summary: string;
  content: string;
  attributes: Record<string, string | string[]>;
  confidence: number;
  evidence: string;
  sourceChapterId: string;
  status: 'pending' | 'confirmed' | 'rejected';
}
```

### 已确认的条目

确认后转为标准 `WikiEntry`：

```typescript
interface WikiEntry {
  // ... existing fields
  confirmedByUser: true;  // 手动或 AI 确认
  confidence: 1;
  sourceChapters: string[];
}
```

## 实现优先级

| 优先级 | 功能 | 预估工作量 |
|--------|------|-----------|
| P0 | AI 提取新条目 | 核心，先做 |
| P1 | AI 补充已有条目 | 依赖 P0 的数据结构 |
| P2 | AI 一致性检查 | 独立模块，可与 P1 并行 |
| P3 | 智能关系发现 | 最后，依赖 P0+P1 的关系数据 |
