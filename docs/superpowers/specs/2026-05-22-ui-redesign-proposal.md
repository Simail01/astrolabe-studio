# 星盘工坊 UI 重新设计方案

> 状态：方案讨论中 | 2026-05-22

## 一、当前问题

当前 UI 是 VS Code 的粗糙复刻——暗色背景 + 蓝色状态栏 + emoji 图标，给作者"在写代码"而非"在创作"的感觉。六个创作阶段的工具全部堆在一个窗口里，任何一个阶段都体验不佳。

## 二、设计原则

1. **渐进式复杂度** — 默认界面只显示创作必需的核心区域，高级功能（Wiki、同人库、分镜）按需展开
2. **上下文感知** — 选中大纲节点 → 显示节点属性；写章节 → 显示 Wiki 参考；做漫画 → 显示素材库
3. **温润的专业感** — 保留暗色主题，暖色调点缀，减少"工程感"的边框和蓝色
4. **每个阶段拥有专属布局** — 不为折中牺牲任何阶段的体验

## 三、配色方案

| 元素 | 当前 (VS Code 感) | 建议 (创作工具感) |
|------|-------------------|-------------------|
| 主背景 | `#1e1e1e` 纯黑灰 | `#1a1b1e` 微暖深灰 |
| 面板 | `#252526` 冷灰 | `#222325` 中性灰 |
| 强调色 | `#007acc` 荧光蓝 | `#c4a35a` 暖金色 |
| 状态栏 | `#007acc` 蓝色 | `#2a2b2e` 融入背景 |
| 选中高亮 | `#094771` 亮蓝 | `#3a3520` 暖色 |
| 成功/警告/错误 | 语义色 | 保留低饱和度版本 |

### 按钮体系

| 级别 | 用途 | 样式 |
|------|------|------|
| 主操作 | AI 生成、保存、确认引入 | 暖金色背景 `#c4a35a` |
| 次要操作 | 取消、展开、补充 | 灰色背景，hover 微亮 |
| 危险操作 | 删除、拒绝 | 暗红色背景 |
| 文字操作 | 编辑、选择 | 无背景，hover 下划线 |

## 四、竞品借鉴

| 产品 | 借鉴点 |
|------|--------|
| **Scrivener** | 左侧 Binder（作品结构树）+ 右侧分屏编辑，一眼看到"作品全貌" |
| **Notion** | 极致简洁，用留白代替边框，顶栏只有标题和面包屑 |
| **Sudowrite** | AI 功能不突兀，以卡片形式展示生成结果 |
| **Ulysses** | 三栏（文库→章节列表→编辑器），文字排版精良，无干扰 |
| **Figma** | 属性面板在右侧，选中什么就显示什么，上下文感知 |

## 五、按阶段切换布局

### 核心概念

六个创作阶段不共享同一个布局——**每个阶段拥有自己专属的窗口布局。**

顶部阶段指示器：

```
星盘工坊    大纲 │ 写作 │ 分镜 │ 漫画 │ 漫剧
            ○    ●     ○     ○     ○
```

点击切换阶段 → 整个主工作区切换到该阶段的专属布局。

### 阶段一：大纲 — 规划模式

```
┌─────────────────────────────────────────────┐
│  菜单栏                          [写作] [*分镜]│
├──────────┬──────────────────────────────────┤
│  作品结构  │      大纲画布（占满主区域）           │
│          │                                  │
│  📁作品A  │   ┌─ 第一卷 ──────────────────┐  │
│  ├─大纲◀  │   │  ├─ 第1章：穿越           │  │
│  ├─角色   │   │  ├─ 第2章：第一桶金        │  │
│  ├─Wiki   │   │  └─ 第3章：天使轮         │  │
│  ├─分镜   │   └──────────────────────────┘  │
│  ├─漫画   │                                  │
│  ├─漫剧   │   [+ 根节点] [AI 生成大纲]         │
│          │                                  │
└──────────┴──────────────────────────────────┘
```

- 左侧：作品结构树（始终可见）
- 主区域：大纲画布全宽展示
- 不显示上下文面板

### 阶段二：写作 — 专注模式

```
┌─────────────────────────────────────────────┐

│ 节  │    章节编辑器          │  ┌────────────┐ │

│ 表  │                      │  │ 军师，性格.. │ │
│    │  第 3 章              │  └────────────┘ │

│    │  诸葛亮坐在茅草屋里，   │  │ 位于...     │ │
│    │  手中的羽扇轻轻摇动    │  └────────────┘ │
│    │  ...                  │                 │
│    │                      │   [AI 参考卡片]  │
│    │  ──────────────────   │                 │
│    │  · 1234 字            │                 │
│    │  AI续写 | Wiki提取     │                 │
│    │                      │                 │
```

- 左侧：章节列表（收缩为窄条）
- 主区域：编辑器居中（最大 700px）
- 右侧：Wiki 参考面板常驻（上下文感知——选中章节自动显示相关 Wiki 条目）

**编辑器改进：**
- 去掉 textarea 可见边框，背景与编辑区融为一体
- 光标颜色改为暖金色
- 工具栏从顶部移到底部（减少对标题区的干扰）

- 不相关面板全部隐藏

### 阶段四：漫画 — 构建模式

```
┌─────────────────────────────────────────────┐
│  菜单栏                          [分镜] [*漫剧]│
├────────────────────┬────────────────────────┤
│   页面画布（65%）    │    素材库 + 工具栏（35%） │
│                    │                        │
│  ┌─────┬─────┬──┐  │  ┌─ 页面模板 ────────┐ │
│  │ 格1  │ 格2  │  │  │ □ 2×2  □ 3×2       │ │
│  │     │     │  │  │ □ 1大3小  □ 自由布局  │ │
│  ├─────┴─────┤  │  │ └──────────────────┘ │
│  │   格3     │  │  │                        │
│  │          │  │  │  ┌─ 分镜素材 ────────┐ │
│  └──────────┘  │  │  │ [镜头1]✓ [镜头4]✓ │ │
│                    │  │ [镜头2]✓ [镜头5]  │ │
│                    │  │ [镜头3]  [镜头6]  │ │
│                    │  │ ...               │ │
│                    │  └──────────────────┘ │
│                    │                        │
│  [生成当前页]       │   ┌─ 对话气泡 ────────┐ │
│                    │   │ □矩形 □椭圆 □云朵  │ │
│                    │   └──────────────────┘ │
├────────────────────┴────────────────────────┤
│  第3章 · 漫画 · 第2页/共5页  [编辑] [预览]      │
└─────────────────────────────────────────────┘
```

**漫画页面交互流程：**

1. 用户从分镜素材库（已生成所有镜头）拖拽镜头到页面画布的格子中
2. 右侧提供页面布局模板（2×2、3×2、1大3小、自由布局）
3. 填充完成后，为每个格子添加对话气泡（矩形/椭圆/云朵/锯齿样式可选）
4. 点"生成当前页" → AI 根据格内镜头数据+气泡位置渲染完整漫画页面
5. 可切换为"预览模式"查看最终效果

**数据模型：**

```typescript
interface ComicPage {
  id: string;
  chapterId: string;
  pageNumber: number;
  layout: '2x2' | '3x2' | '1big3small' | 'full' | 'custom';
  panels: ComicPanel[];
}

interface ComicPanel {
  id: string;
  shotId: string;
  imageUrl?: string;
  position: { row: number; col: number; rowSpan: number; colSpan: number };
  bubbles: SpeechBubble[];
}

interface SpeechBubble {
  id: string;
  text: string;
  style: 'rect' | 'ellipse' | 'cloud' | 'jagged' | 'none';
  speaker: string;
}
```

## 七、统一组件风格

| 组件 | 当前 | 建议 |
|------|------|------|
| 图标 | Emoji（大小不一） | 统一 16px Unicode 符号或 SVG |
| 按钮 | 蓝色/灰色/棕色/红色混用 | 三档统一（金色主操作/灰色次要/暗红危险） |
| 状态提示 | 横幅式（整行） | Toast 风格（右下角浮层，3秒消失） |
| 对话框 | 直接弹出 | 微动画 + 背景模糊 |
| 列表选中 | 整行纯色高亮 | 左侧 2px 金色竖线指示 |
| 边框 | 到处都是 | 大量减少，用间距区分 |
| 滚动条 | 系统默认 | 统一 6px 宽度，暗色 |
