\## 一、产品哲学



\### 1. 产品定位



“星盘工坊 v2”是一个创作型生产力工具，面向\*\*小说、漫画、漫剧创作者\*\*，核心价值是：



1\. \*\*沉浸创作\*\*：用户在任何阶段都能专注故事创作，而不是工具操作。

2\. \*\*全流程覆盖\*\*：从大纲→写作→分镜→漫画→漫剧，形成一体化创作管线。

3\. \*\*AI 协作\*\*：AI 是创作搭档，而非按钮功能。提供智能建议、续写、逻辑检测。

4\. \*\*认知贴合\*\*：信息架构与创作者心智模型贴合，而不是按工程功能拆解。



\### 2. 用户心智模型



创作者思考方式：



\* \*\*世界 → 剧情 → 输出\*\*

\* 时常在阶段间跳跃（写作→大纲→漫画→写作）

\* 需要即时参考（Wiki / 角色 / 地点 / 分镜）

\* 关注节奏与情绪（剧情高潮、章节进度）

\* AI 不应该打断沉浸，而是辅助决策



\### 3. 设计原则



\* \*\*渐进式复杂度\*\*：默认简洁，按需展开。

\* \*\*上下文感知\*\*：选中元素，显示对应属性。

\* \*\*沉浸感\*\*：去掉工程感，暖色点缀，专注创作。

\* \*\*阶段独立但连续\*\*：布局优化，允许跨阶段快速切换。

\## 五、创作模块



\### 1. 大纲（规划模式）



```

┌───────────────────────────┐

│  大纲画布 (可切换树 / 卡片) │

├───────────────┬───────────┤

│  左侧树       │ 主区域画布 │

│  世界观/剧情  │ 卷/章可拖拽 │

└───────────────┴───────────┘

```



特点：



\* \*\*Tree ↔ Story Map 双视图\*\*：线性 + 空间化卡片

\* 拖拽章节顺序

\* AI 提供结构优化建议（右下浮层悬浮）

\* 当前选中节点高亮 + 属性面板



\### 2. 写作（沉浸模式）



```

┌─────────────────────────────────────┐

│         顶部标题栏                   │

├──────────┬──────────────────────────┤

│ 左侧章节 │ 主编辑区 (700px 居中)      │

│ 列表     │ ┌─────────────────────┐ │

│          │ │ 今天天气闷热…        │ │

│          │ │ 光标：暖金色          │ │

│          │ └─────────────────────┘ │

│          │ 下方悬浮 AI 工具气泡     │

│          │ \[续写] \[改文风] \[增强情绪]│

├──────────┴──────────────────────────┤

│ 底部状态栏：字数 / AI 状态 / 当前章节 │

└─────────────────────────────────────┘

```



特点：



\* 编辑器无边框、背景融入主区域

\* AI 悬浮气泡，与文字上下文绑定

\* Wiki / 世界观 / 角色信息默认折叠，按需展开

\* 支持 Markdown + 富文本混排

\## 六、视觉化模块

```

┌─────────────────────────────────────┐

```



\* 页面格子支持拖拽镜头

\* 右侧抽屉切换模板 / 素材 / 对话

\* AI 渲染按钮仅显示在抽屉顶部

\* 支持自由布局 + 预览模式

\### 1. 悬浮 AI



\* 与文字或镜头绑定

\* 浮动气泡或右下角浮层

\* 功能：



&#x20; \* 续写

&#x20; \* 改文风

\* 选中章节 / 镜头 → 显示相关角色/地点/Wiki

\* 默认折叠，点击展开

\* 提高沉浸感

\## 十二、总结



\* \*\*统一认知模型\*\*：世界→剧情→输出

\* \*\*Workspace + Context-aware\*\*：用户操作自然流畅

\* \*\*沉浸写作\*\*：AI 不打断、悬浮辅助

\* \*\*漫画/漫剧流程\*\*：抽屉工具 + 页面画布 + 渲染按钮

\* \*\*设计系统统一\*\*：颜色、光标、动效、滚动条

\* \*\*组件树清晰\*\*：React + Tauri + TypeScript 可直接实现
