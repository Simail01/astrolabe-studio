# Wiki 知识库系统 — 设计规格

> 子系统 #2 | 状态：已确认 | 2026-05-21

## 定位

每个作品的"外置大脑"。解决长篇小说后期角色关系混乱、剧情矛盾、设定遗忘的问题。结构化存储作品中的一切信息，AI 在创作时自动从 Wiki 检索相关上下文注入 prompt。

**不导出。** Wiki 是作者的私人创作笔记。

## 核心设计

```
        自动提取                 手动录入
           │                       │
           ▼                       ▼
    ┌─────────────────────────────────────┐
    │            Wiki 引擎                 │
    │                                     │
    │  ┌─────────┐  ┌─────────┐          │
    │  │ 实体索引 │  │ 关系图谱 │          │
    │  │ (倒排)   │  │ (图结构) │          │
    │  └─────────┘  └─────────┘          │
    │                                     │
    └─────────────────────────────────────┘
              │
              ▼
      AI 上下文注入（创作时自动检索相关内容）
```

## Wiki 条目类型

```
Wiki 条目
├── 角色 (Person)
│   ├── 姓名、别名、身份/职业
│   ├── 外貌描述、性格特征、能力/技能
│   ├── 人际关系 (→ 关联其他角色条目)
│   ├── 首次出场章节
│   └── 角色弧光 (从→到 的成长轨迹)
│
├── 地点 (Location)
│   ├── 名称、地理位置、环境描述
│   ├── 所属势力
│   └── 相关事件
│
├── 组织/势力 (Faction)
│   ├── 名称、宗旨/目标
│   ├── 成员 (→ 关联角色条目)
│   ├── 势力范围、历史
│
├── 物品/道具 (Item)
│   ├── 名称、类型 (武器/法宝/药剂/…)
│   ├── 能力/效果、来源
│   └── 持有者 (→ 关联角色条目)
│
├── 事件 (Event)
│   ├── 名称、发生时间 (作品内时间线)
│   ├── 参与角色、发生地点
│   ├── 起因/经过/结果
│   └── 对后续剧情的影响
│
├── 规则/设定 (Rule)
│   ├── 世界观规则 (力量体系、魔法规则、科技水平…)
│   ├── 社会规则 (法律、习俗、等级…)
│   └── 约束条件
│
└── 时间线 (Timeline)
    ├── 章节 → 事件映射
    └── 角色出场时间线
```

## 数据模型

```typescript
interface WikiEntry {
  id: string;
  type: 'person' | 'location' | 'faction' | 'item' | 'event' | 'rule';
  title: string;
  aliases: string[];
  summary: string;           // AI 生成的摘要，用于快速注入
  content: string;           // 完整 Markdown 描述
  attributes: Record<string, string | string[]>;
  relations: WikiRelation[];
  sourceChapters: string[];  // 信息来源自哪些章节
  confidence: number;        // AI 推断置信度 0-1，人工确认后为 1
  createdAt: string;
  updatedAt: string;
  confirmedByUser: boolean;
}

interface WikiRelation {
  targetId: string;
  relationType: string;      // 父子/师徒/敌对/从属/…
  description: string;
}
```

## AI 自动提取流程

```
作者写完一章 → 触发 "Wiki 提取"
                    │
                    ▼
          将章节全文 + 现有 Wiki 发送给 DeepSeek
                    │
                    ▼
          AI 分析输出：
          1. 新出现的角色/地点/物品
          2. 已有角色的新信息（能力变化、新关系…）
          3. 新发生的事件
          4. 可能暗示的新规则/设定
                    │
                    ▼
          以 "建议" 形式呈现给作者
          作者可以：✓ 确认 / ✎ 修改后确认 / ✕ 拒绝
                    │
                    ▼
          确认的条目写入 wiki/ 目录
```

## 上下文注入机制

当 AI 创作新章节时，自动检索并注入相关 Wiki 内容：

```
1. AI 收到 "续写章节" 请求
2. Wiki 引擎检索：
   a. 当前章节涉及哪些角色？→ 检索这些角色的 Wiki 条目
   b. 当前场景在哪个地点？→ 检索该地点条目
   c. 最近的剧情节点涉及哪些事件？→ 检索相关事件
   d. 当前剧情是否触发了某些规则？→ 检索相关规则
3. 将检索结果组装成上下文，注入 System Prompt
4. AI 在创作时就能"记住"所有相关设定
```

## 关系图谱可视化

前端使用力导向图（D3.js force layout 或 react-flow）展示条目间关系。角色、地点、事件通过关系连线可视化，作者可以直观看到 "谁认识谁"、"在哪里发生了什么"。

## AI 提取 Prompt 设计

```
你是一位小说编辑助理。请分析以下章节内容，提取新信息并更新知识库。

## 现有知识库
{{existingWikiSummary}}

## 新章节内容
{{chapterContent}}

## 请提取
1. 新角色：姓名、身份、外貌、性格、能力
2. 已有角色的新信息：状态变化、新能力、新关系
3. 新地点：描述、所属势力
4. 新事件：参与者、时间、地点、影响
5. 新规则/设定：世界观规则的补充或修改

## 输出格式
返回 JSON，对于每项信息标注：
- confidence: 你的确定程度 (0-1)
- evidence: 原文中支持此推断的句子
```

## 导出策略

Wiki **不导出**。它是作者的创作工具和私人笔记，不是作品的产品形态。
